Productor/Consumidor

Sincronizacion necesaria

A fin de evitar condiciones de carrera (Race Condition), es necesario sincronizar el productor con el conusumidor evitando que accedan o modifiquen la cola al mismo tiempo asi no generan una inconsistencia en los datos.

¿Qué pasa si el productor y el consumidor acceden libremente a la cola?

Si el productor y el consumidor acceden a la cola sin sincronización, puede producirse una condición de carrera (race condition). Esto ocurre porque ambos hilos modifican o leen la estructura al mismo tiempo, generando comportamientos impredecibles. Como consecuencia, pueden perderse datos, consumirse elementos incorrectos, duplicarse tareas o incluso producirse errores al intentar extraer elementos de una cola vacía. La sincronización es necesaria para garantizar que las operaciones sobre la cola se realicen de forma segura y consistente entre los hilos.

Cola thread-safe

Usando una cola thread-safe obtenemos un código mas limpio y legible, y además reduce la posibilidad de errores como deadlocks o race conditions. También nos elimina la necesidad de gestionar locks manualmente.

Ejercicio

Implementar un sistema donde:

  • Un productor genera números del 1 al 20.
  • Un consumidor toma esos números y los procesa.
  • Debe existir una cola compartida.
  • El productor no debe agregar datos si la cola esta llena.
  • El consumidor no debe consumir si la cola esta vacía.
